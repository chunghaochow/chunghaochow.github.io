回到add前，即当前版本工作区变动还在

回到commit前, add 后的状态，即回到暂存状态。场景：commit时候写错备注，可以reset –soft 回去，重新commit

回到指定版本，工作区、暂存区都是干净的。

reset 指定文件

与整个版本的回滚有所不同，指定文件回滚，版本号并不会改变，只是将指定commit_id 的内容放入暂存区，再将当前版本的改动添加到工作区。所以git status 可以看到以下两个状态。

场景：例如dev.md，我希望这个文件的内容回到上一个版本，git reset HEAD^ dev.md，然后git checkout – dev.md，丢弃当前版本的改动，此时git status, 变为以下状态

那么这就OK了，你可以提交版本了。把指定版本号的改动作为新的commit 提交。
